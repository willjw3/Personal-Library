Features

  1. Nothing from user's website will be cached in their client as a security measure.
  2. User will see that the site is powered by 'PHP 4.2.0' (even though it isn't) as a security measure.
  3. User can POST a title to /api/books to add a book.
  4. User can GET /api/books to retrieve a list of all books.
  5. User can GET /api/books/{_id} to retrieve a single book.
  6. User can POST a comment to /api/books/{_id} to add a comment to a book.
  7. User can DELETE a book from the collection.
  8. If user tries to request a book that doesn't exist they will get a 'no book exists' message.
  9. User can send a DELETE request to /api/books to delete all books in the database.
  10. All functional tests are complete and passing.

This is a freeCodeCamp Information Security and Quality Assurance project
